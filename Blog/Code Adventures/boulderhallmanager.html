Boulder Hall Manager - Rotation Handles, Command Pattern & Outline Renderer

Posted on 5-9-2023 at 03:12 am

It's 3 am, my circadian rhyth begins to shift, and I have some exciting updates to share with you.

First, I've added "Undo" functionality to the program. To make this work seamlessly, I had to refactor my existing code to follow the Command Pattern. It took some time, but it's made the code more efficient and easier to expand. While I've implemented "Undo," I haven't yet worked on the "Redo" function

Second, I've added rotation handles, fulfilling a promise (to whom, I'm not quite sure!). This feature was both straightforward and rewarding to implement. I don't have much to elaborate on here—it simply works as expected.

Third, I've added the ability to remove existing holds or grips. This was easier to implement than ever, thanks to the newly incorporated Command Pattern in the code.

Last but not least, I've rolled out support for multiple colored outlines. I made adjustments to the shader to accommodate this feature. The objects requiring outlines are passed through an UV channel, with their color in the screen render determining the outline color. Since no shadow data is passed through during these render calls, the meshes in the screen render are displayed in flat color which makes the line color consistent.

Boulder Hall Manager - Path Editor & Outline Renderer

Posted on 4-9-2023 at 02:22 am

Path Editor

Hello, it's 2 am again, so you know what that means! Time to post an update about the latest changes I made to my management game. I made a programming interface for seperate paths, which are visualised by the outline. In the next update I will probably add some handles for editing already placed grips.

Outline Shader

As you can see I also implemented my own Outline shader, which took some hours to make. I don't have alot of experience creating shaders so this was a fun way to learn more about them and the unity render pipeline. In short I created a custom post processing effect and passed a render texture which contains the meshes of objects that need to be rendered. After that I do some edge detection in the shader and sample 4 points around the center point of the shader pass. The line width determines how far those points are from the center. For now I'm very happy with the results and what I learned
